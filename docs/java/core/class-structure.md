# Устройство класса

Класс — это чертёж: он описывает поля (состояние), методы (поведение)
и конструкторы (как создать объект). Здесь — то, что нужно понимать про его
устройство: модификаторы доступа, `static`, `final`, порядок инициализации,
вложенные классы и `enum`.

## Конструкторы

Конструктор — код, который приводит новый объект в корректное состояние.

```java
public class Order {
    private final String id;
    private final List<Item> items;

    public Order(String id, List<Item> items) {
        this.id = id;
        this.items = items;
    }

    // конструкторы можно перегружать и вызывать один из другого через this(...)
    public Order(String id) {
        this(id, new ArrayList<>());
    }
}
```

Что нужно помнить:

- Если не написать ни одного конструктора, компилятор создаст **конструктор
  по умолчанию** — пустой, без аргументов. Но как только появляется хоть один
  свой, дефолтный исчезает.
- `this(...)` — вызов другого конструктора этого же класса, `super(...)` —
  конструктора родителя. И тот и другой могут стоять только **первой строкой**.
- Конструктор не должен делать тяжёлую работу (запросы в сеть, чтение файлов) —
  его дело собрать объект. Тяжёлое — в отдельные методы или фабрики.

## Порядок инициализации

Классический вопрос «что выведет этот код» строится на порядке, в котором
инициализируется класс и объект:

1. **Статические** поля и статические блоки `static { ... }` — один раз,
   при первой загрузке класса, сверху вниз.
2. При каждом `new`: поля экземпляра и блоки инициализации `{ ... }` —
   сверху вниз.
3. Конструктор.

При наследовании сначала полностью инициализируется родительская часть:
статика родителя → статика наследника → поля и конструктор родителя →
поля и конструктор наследника.

!!! warning "Ловушка: виртуальный вызов из конструктора"
    Если конструктор родителя вызывает метод, переопределённый в наследнике,
    метод наследника выполнится **до** инициализации полей наследника —
    и увидит в них `null`/нули. Поэтому из конструктора не вызывают
    переопределяемые методы.

## Модификаторы доступа

Четыре уровня, от закрытого к открытому:

| Модификатор | Класс | Пакет | Наследник | Все |
|---|:---:|:---:|:---:|:---:|
| `private` | ✔ | | | |
| *(без модификатора)* — package-private | ✔ | ✔ | | |
| `protected` | ✔ | ✔ | ✔ | |
| `public` | ✔ | ✔ | ✔ | ✔ |

Практика проста: **всё максимально закрыто**. Поля — `private`, методы —
`private`, пока кому-то снаружи не понадобятся. Чем меньше открыто, тем меньше
кода зависит от внутренностей класса и тем безопаснее его менять.
Package-private полезен для классов, которые являются деталью реализации
пакета, — наружу торчит один публичный класс, остальные скрыты.

## static

`static` означает «принадлежит классу, а не объекту». Статическое поле —
одно на всех, статический метод вызывается без объекта:

```java
public class IdGenerator {
    private static final AtomicLong counter = new AtomicLong(); // одно на всех
    public static long next() { return counter.incrementAndGet(); }
}
```

Что важно понимать:

- Статический метод **не видит** поля экземпляра и `this` — не с чем работать,
  объекта нет.
- Статические методы не участвуют в полиморфизме: их нельзя переопределить,
  только скрыть, и выбор метода происходит по типу ссылки при компиляции.
- Уместное применение: утилитные методы (`Math.max`, `Collections.sort`),
  константы, фабричные методы (`List.of`, `Optional.of`).
- Изменяемое статическое состояние — источник проблем: это фактически глобальная
  переменная, её видят все потоки сразу. В Spring-приложениях общее состояние
  живёт в бинах, а не в статике.

## final

Одно слово — три разных смысла в зависимости от места:

- **Поле/переменная**: присвоить можно один раз. Для ссылки это значит
  «ссылка не поменяется», но сам объект по ссылке менять можно —
  `final List` не мешает делать `add()`.
- **Метод**: нельзя переопределить в наследнике.
- **Класс**: нельзя наследовать (`String`, `Integer` — `final`).

Хорошая привычка — делать поля классов `final` везде, где возможно: объект
с неизменяемыми полями проще понимать и безопасно отдавать между потоками.

## Вложенные классы

Класс можно объявить внутри другого класса. Варианты:

- **Статический вложенный** (`static class`) — просто класс, живущий в
  пространстве имён внешнего. Не держит ссылку на внешний объект.
  Это вариант по умолчанию, если вложенный класс нужен.
- **Внутренний** (без `static`) — каждый экземпляр привязан к экземпляру
  внешнего класса и держит на него скрытую ссылку. Из-за этой ссылки внешний
  объект не может быть собран сборщиком мусора, пока жив внутренний, —
  источник утечек памяти.
- **Анонимный** — класс без имени, объявленный прямо в месте использования.
  До Java 8 так передавали поведение (`new Runnable() { ... }`); сейчас для
  функциональных интерфейсов используют лямбды, а анонимные классы остались
  для случаев, когда нужно переопределить несколько методов.

## enum

`enum` — класс с фиксированным набором экземпляров, известным при компиляции.
Каждое значение — единственный объект, поэтому их безопасно сравнивать через `==`.

```java
public enum OrderStatus {
    NEW("Новый"),
    PAID("Оплачен"),
    SHIPPED("Отправлен");

    private final String title;

    OrderStatus(String title) { this.title = title; }
    public String getTitle() { return title; }
}
```

Возможности, которые стоит знать:

- `enum` может иметь поля, конструкторы (только `private`) и методы —
  это полноценный класс, наследующий `java.lang.Enum`.
- `values()` — массив всех значений, `valueOf("PAID")` — значение по имени
  (кидает исключение, если имени нет).
- В `switch` по enum перечисляются значения без имени типа: `case PAID ->`.
- Отлично подходит для статусов, типов, стратегий. Метод можно даже
  переопределить на каждое значение — получается компактный вариант
  паттерна «стратегия».

## Как ответить на интервью

Коротко: класс собирается из полей, методов и конструкторов; инициализация идёт
в порядке «статика → поля экземпляра → конструктор», при наследовании — сначала
родитель. Доступ по умолчанию максимально закрытый. `static` — принадлежность
классу и отказ от полиморфизма, `final` — запрет изменения/переопределения/
наследования. `enum` — класс с фиксированным набором экземпляров, стандартный
выбор для статусов.
