# Set

`Set` — множество: дубликаты запрещены, `add` уже существующего элемента
возвращает `false` и ничего не меняет. Главная операция — **проверка
принадлежности** `contains`, и ради неё Set и выбирают.

## Реализации — зеркало Map

Каждая реализация `Set` внутри буквально построена на соответствующей `Map`
(элемент — ключ, значение — заглушка), поэтому вся механика — та же:

| | Внутри | `add`/`contains` | Порядок обхода |
|---|---|---|---|
| `HashSet` | `HashMap` | O(1) | нет |
| `LinkedHashSet` | `LinkedHashMap` | O(1) | порядок вставки |
| `TreeSet` | `TreeMap` | O(log n) | по сортировке |

Следствия те же, что у Map:

- `HashSet` требует согласованных `equals`/`hashCode` и неизменности элементов.
- `TreeSet` сравнивает элементы **только** через `compareTo`/`Comparator`.
  Отсюда сюрприз: два объекта, неравных по `equals`, но равных по `compareTo`,
  для TreeSet — «один и тот же элемент», второй не добавится.
- `TreeSet` даёт навигацию: `first()`, `last()`, `floor(x)`, `headSet`,
  `subSet` — когда нужны «ближайшие» элементы или диапазоны.

## Когда использовать Set

- **Дедупликация**: `new HashSet<>(list)` — самый короткий способ убрать
  дубликаты (с потерей порядка; с сохранением — `LinkedHashSet` или
  `stream().distinct()`).
- **Частые проверки «есть ли»**: `contains` у `HashSet` — O(1) против O(n)
  у списка. Если в коде `list.contains(...)` вызывается в цикле —
  это готовая квадратичная сложность, и замена списка на Set чинит
  производительность одной строкой.
- **Семантика уникальности**: тип `Set<Role>` сам документирует, что ролей
  без повторов, — это лучше комментария к `List`.

Операции над множествами делаются прямо через API коллекций — на месте,
изменяя текущее множество:

```java
a.retainAll(b);  // пересечение: осталось то, что есть в обоих
a.addAll(b);     // объединение
a.removeAll(b);  // разность
```

## EnumSet: множество enum-значений

Если элементы — значения одного `enum`, вместо `HashSet` используется
`EnumSet`: внутри это битовая маска в `long`, поэтому он компактнее
и быстрее любого хеша:

```java
Set<DayOfWeek> weekend = EnumSet.of(SATURDAY, SUNDAY);
Set<DayOfWeek> workdays = EnumSet.complementOf(weekend);
```

Аналогично для ключей-enum в Map есть `EnumMap` (внутри — массив по ordinal).

## Как ответить на интервью

Коротко: Set хранит уникальные элементы, а его реализации построены поверх
одноимённых Map — `HashSet` даёт O(1) `contains` и требует корректных
`equals`/`hashCode`, `LinkedHashSet` помнит порядок вставки, `TreeSet` держит
элементы отсортированными и сравнивает только через `compareTo`. Типовые
применения — дедупликация и быстрые проверки принадлежности: `contains`
по списку в цикле — сигнал заменить список на Set. Для enum — `EnumSet`
на битовой маске.
