# Агрегация и группировка

Агрегация — сведение множества строк к одному значению: сумма, количество,
среднее. `GROUP BY` делает это по группам.

## Агрегатные функции

- `count(*)` — число строк; `count(col)` — число строк, где `col` не `NULL`;
  `count(DISTINCT col)` — число уникальных значений.
- `sum`, `avg`, `min`, `max` — по числовому столбцу, `NULL` пропускаются.
- `string_agg(col, ', ')` (PostgreSQL) — склеить значения группы в строку;
  `array_agg` — собрать в массив.

## GROUP BY

`GROUP BY` разбивает строки на группы по значениям указанных столбцов и
считает агрегат для каждой:

```sql
SELECT   user_id, count(*) AS cnt, sum(amount) AS total
FROM     orders
GROUP BY user_id;
```

Ключевое правило: **в `SELECT` при `GROUP BY` можно ставить только
сгруппированные столбцы и агрегаты.** Нельзя `SELECT user_id, product_id`,
группируя лишь по `user_id`, — у группы много разных `product_id`, СУБД не
знает, какой показать (PostgreSQL выдаст ошибку; MySQL исторически молча
возвращал случайный — источник багов).

## WHERE vs HAVING

Разница вытекает из порядка обработки:

- `WHERE` — фильтр **до** группировки, по отдельным строкам.
- `HAVING` — фильтр **после** группировки, по значению агрегата.

```sql
SELECT   user_id, count(*) AS cnt
FROM     orders
WHERE    status = 'paid'      -- сначала отбираем оплаченные строки
GROUP BY user_id
HAVING   count(*) > 3;        -- потом группы, где таких больше трёх
```

`WHERE count(*) > 3` — ошибка: на этапе `WHERE` агрегата ещё нет.

## Оконные функции как альтернатива

Когда нужен агрегат **рядом с каждой строкой**, а не вместо неё, — оконные
функции вместо `GROUP BY`:

```sql
SELECT id, user_id, amount,
       sum(amount) OVER (PARTITION BY user_id) AS user_total
FROM   orders;
```

`GROUP BY` схлопнул бы заказы в одну строку на пользователя; окно оставляет
все заказы, добавляя к каждому сумму по его пользователю. Так же считают
running total (`ORDER BY` внутри `OVER`), ранги, «предыдущую строку» (`lag`).

## Как ответить на интервью

Коротко: агрегатные функции сводят строки к одному значению, `GROUP BY`
делает это по группам. Главное правило — в `SELECT` только столбцы из
`GROUP BY` и агрегаты. `WHERE` фильтрует строки до группировки, `HAVING` —
группы после, по агрегату. Если агрегат нужен рядом с каждой строкой, а не
вместо неё — оконная функция с `OVER (PARTITION BY ...)`, она не схлопывает
строки.
