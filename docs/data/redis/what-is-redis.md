# Что такое Redis

Redis — хранилище данных типа «ключ-значение», работающее **в оперативной
памяти**. Из-за этого он очень быстрый (отклик — доли миллисекунды) и потому
чаще всего используется как кэш перед основной базой. Но это не только кэш —
у него богатый набор структур данных и много применений.

## Ключевые свойства

- **In-memory** — данные лежат в RAM, отсюда скорость и отсюда же
  ограничение: объём ограничен памятью, а при падении без специальных мер
  данные теряются (см. тему про персистентность).
- **Однопоточная обработка команд.** Redis выполняет команды **по одной**, в
  одном потоке. Это не слабость: нет блокировок и гонок между командами,
  каждая команда атомарна. Но следствие — **тяжёлая команда блокирует всех**
  (например, `KEYS *` на большой базе), таких избегают.
- **Ключи с TTL** — любому ключу можно задать время жизни, по истечении Redis
  сам его удалит. Основа кэширования.

## Структуры данных

Redis хранит не только строки — в этом его сила:

| Тип | Что это | Типичное применение |
|---|---|---|
| String | строка/число | кэш значений, счётчики (`INCR`) |
| Hash | словарь полей | объект (профиль) по полям |
| List | список | очередь, лента |
| Set | множество | уникальные, пересечения |
| Sorted Set | множество с весом | рейтинги, лидерборды, отложенные задачи |
| прочее | Bitmap, HyperLogLog, Stream, Geo | счётчики уникумов, потоки событий |

`INCR` (атомарный инкремент), `EXPIRE` (TTL), Sorted Set для рейтингов —
частые примеры, которые полезно назвать.

## Как это выглядит из приложения

Redis — отдельный сервер, приложение общается с ним по сети через клиент
(в Java — Lettuce/Jedis, в Spring — `RedisTemplate`/Spring Cache). Обращение
к Redis — это сетевой вызов, пусть и быстрый; поэтому его тоже нельзя делать
в цикле по одному там, где можно пачкой (pipeline/`MGET`).

## Как ответить на интервью

Коротко: Redis — быстрое in-memory хранилище ключ-значение, чаще всего кэш
перед базой (отклик — доли миллисекунды). Однопоточный: команды выполняются
по одной и потому атомарны, но тяжёлая команда блокирует всех. Любому ключу
можно задать TTL — основа кэша. Помимо строк умеет hash, list, set, sorted
set — за счёт этого используется не только как кэш: счётчики через `INCR`,
рейтинги через sorted set, очереди через list. Из приложения — отдельный
сервер, доступ по сети через клиент.
