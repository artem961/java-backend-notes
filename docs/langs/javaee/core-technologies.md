# Основные технологии

Обзор главных спецификаций Jakarta EE — что делает каждая и где
с ней сталкивается Spring-разработчик. Глубоко копать их по отдельности
не нужно; важно понимать роль каждой и связь с современным стеком.

## Servlet — фундамент всей веб-Java

**Сервлет** — класс, обрабатывающий HTTP-запросы: контейнер сервлетов
(Tomcat, Jetty) принимает соединения, разбирает HTTP и вызывает
`service(request, response)` подходящего сервлета. Плюс **фильтры**
(сквозная обработка до/после сервлета) и **слушатели** (события жизненного
цикла).

Почему это важно: **Spring MVC — это один большой сервлет**
(`DispatcherServlet`), который принимает все запросы и раздаёт их
контроллерам, а Spring Security — цепочка сервлетных фильтров. Понимание
модели «контейнер → фильтры → сервлет» — ключ к вопросам про путь запроса
в Spring.

## JAX-RS — REST по стандарту

Стандартный API для REST-сервисов — то, что в Spring делает
`@RestController`:

```java
@Path("/users")
public class UserResource {
    @GET
    @Path("/{id}")
    @Produces(MediaType.APPLICATION_JSON)
    public UserDto get(@PathParam("id") long id) { ... }
}
```

Spring использует **свои** аннотации (`@GetMapping`, `@PathVariable`) —
исторически Spring MVC появился раньше и стандарту не следует. JAX-RS
встречается в Quarkus, Jakarta EE-приложениях и старых сервисах на Jersey.

## CDI — стандартное внедрение зависимостей

Contexts and Dependency Injection — стандарт DI: `@Inject` для внедрения,
`@ApplicationScoped`/`@RequestScoped` для скоупов, квалификаторы,
события. Концептуально то же, что Spring DI (Spring даже понимает
`@Inject`), но стандартизировано. Основной DI в Quarkus.

## JPA — персистентность

Самая живая спецификация: стандарт ORM — сущности (`@Entity`),
`EntityManager`, JPQL. Реализация де-факто — Hibernate. В Spring
используется через Spring Data JPA; подробный разбор — в блоке «Данные».

## EJB — легаси, которое заменил Spring

Enterprise JavaBeans — исторический стандарт серверных компонентов:
контейнер давал бинам транзакции, безопасность, удалённые вызовы, пулинг.
Ранние версии были тяжёлыми и многословными — **именно как реакция на EJB
родился Spring**, предложив те же сервисы (транзакции, DI) на обычных
POJO-классах. Сегодня EJB — маркер легаси; в Jakarta EE его роль
де-факто перешла к CDI.

## Остальное, что стоит узнавать по имени

- **JMS** — стандарт очередей сообщений (реализации — ActiveMQ, Artemis);
  концептуальный предок того, что сейчас чаще делают Kafka/RabbitMQ.
- **Bean Validation** — декларативная валидация: `@NotNull`, `@Size`,
  `@Email` на полях + `@Valid` на входе. Живёт в каждом Spring-проекте,
  реализация — Hibernate Validator.
- **JTA** — распределённые транзакции; в Spring за этим стоит
  `PlatformTransactionManager`.
- **JSF** — серверный UI-фреймворк (страницы из компонентов на сервере);
  вытеснен SPA-фронтендами, встречается в легаси.

## Как ответить на интервью

Коротко: главные спецификации — Servlet (модель обработки HTTP; Spring MVC —
это `DispatcherServlet`, Spring Security — сервлетные фильтры), JPA
(стандарт ORM, реализация — Hibernate), JAX-RS (стандартные REST-аннотации,
аналог `@RestController`), CDI (стандартный DI, `@Inject`), Bean Validation
(`@NotNull`/`@Valid`). EJB — исторический стандарт компонентов, в ответ
на тяжесть которого появился Spring; сейчас легаси. Знание ролей этих
спецификаций объясняет, на чём стоит современный Spring-стек.
