# Синтаксис и отличия от Java

Kotlin — язык от JetBrains, работающий на JVM. Компилируется в тот же байткод,
что и Java, поэтому полностью с ней совместим: Kotlin-код вызывает Java-библиотеки
и наоборот, оба языка могут жить в одном проекте. Идея Kotlin — убрать
многословность Java и закрыть её исторические проблемы (прежде всего `null`),
не ломая совместимость с экосистемой.

## Базовый синтаксис на контрасте

```kotlin
// переменные: val — неизменяемая (аналог final), var — изменяемая
val name = "Артём"        // тип выводится автоматически
var count: Int = 0        // тип можно указать явно

// функция: fun, тип возврата после сигнатуры
fun sum(a: Int, b: Int): Int {
    return a + b
}

// однострочная функция — expression body
fun sum2(a: Int, b: Int) = a + b
```

Ключевые отличия, которые видны сразу:

- **Точка с запятой не нужна.**
- **Вывод типов**: компилятор сам понимает тип из значения, писать его
  обязательно только там, где вывести нельзя.
- **`val` по умолчанию**: идиоматичный Kotlin-код почти весь на неизменяемых
  переменных, `var` — осознанное исключение. В Java `final` пишут редко,
  потому что он добавляет шум; в Kotlin неизменяемость дешевле изменяемости.
- **Всё — выражение**: `if`, `when`, `try` возвращают значение, поэтому
  тернарный оператор не нужен: `val max = if (a > b) a else b`.

## `when` вместо `switch`

`when` — сильно улучшенный `switch`: без `break`, с диапазонами, типами
и произвольными условиями:

```kotlin
val description = when (code) {
    200 -> "OK"
    in 400..499 -> "ошибка клиента"
    in 500..599 -> "ошибка сервера"
    else -> "что-то ещё"
}
```

В современной Java появился похожий switch-expression — Kotlin имел это
с самого начала, и `when` до сих пор гибче (условия без аргумента, smart cast).

## Мелочи, экономящие код

- **Строковые шаблоны**: `"Пользователь $name, заказов: ${orders.size}"` —
  вместо конкатенации.
- **Параметры по умолчанию и именованные аргументы** — закрывают большинство
  случаев, где в Java городят перегрузки и билдеры:

```kotlin
fun connect(host: String, port: Int = 5432, ssl: Boolean = false) { ... }

connect("db.local")                    // порт и ssl по умолчанию
connect("db.local", ssl = true)        // именованный аргумент, порт пропущен
```

- **Функции верхнего уровня**: функция может жить в файле без класса —
  не нужно заворачивать утилиты в `class StringUtils`.
- **Extension-функции**: можно «добавить» метод чужому классу, не трогая его.
  Под капотом это обычная статическая функция, но вызывается как метод:

```kotlin
fun String.isValidEmail(): Boolean = contains("@") && contains(".")

"user@mail.ru".isValidEmail()   // выглядит как метод String
```

На extension-функциях построена значительная часть стандартной библиотеки
Kotlin и Kotlin-API Spring.

## Совместимость с Java

Главное архитектурное свойство: **Kotlin и Java взаимозаменяемы на уровне
байткода**. Практически это значит:

- Kotlin-класс может наследовать Java-класс, реализовать Java-интерфейс,
  использовать любую Java-библиотеку (Spring, Hibernate, Jackson).
- Миграция проекта возможна постепенно — файл за файлом, оба языка
  компилируются в один проект.
- Из Java Kotlin-код тоже виден: функции верхнего уровня становятся
  статическими методами класса `ИмяФайлаKt`.

## Как ответить на интервью

Коротко: Kotlin — JVM-язык, полностью совместимый с Java, нацеленный на
лаконичность и безопасность. Основное: `val`/`var` с выводом типов и упором
на неизменяемость, всё — выражение (`if`/`when` возвращают значения),
`when` мощнее `switch`, строковые шаблоны, параметры по умолчанию вместо
перегрузок, extension-функции. Java-экосистема доступна целиком, миграция
возможна постепенная — поэтому Kotlin легко заходит в существующие
Java-проекты.
