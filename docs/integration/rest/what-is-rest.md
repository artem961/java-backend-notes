# Что такое REST

REST — это архитектурный **стиль** для API поверх HTTP, а не протокол и не
стандарт. Идея: обращаемся к **ресурсам** по URL, действия над ними выражаем
**HTTP-методами**, а состояние передаём в представлениях (обычно JSON).

## Ключевые принципы

- **Ресурсы и URL** — сущности (`/users`, `/users/42/orders`), а не действия.
  URL описывает «что», метод — «что сделать».
- **Единообразный интерфейс** — стандартные методы (`GET`/`POST`/`PUT`/
  `PATCH`/`DELETE`) и коды статуса вместо своих команд.
- **Stateless** — каждый запрос самодостаточен, сервер не хранит клиентскую
  сессию между запросами; всё нужное (токен и т.п.) — в самом запросе.
- **Представления** — один ресурс можно отдавать в разных форматах (JSON,
  XML); клиент и сервер договариваются через `Accept`/`Content-Type`.

## Ресурсо-ориентированный URL

Плохо (действие в URL) → хорошо (ресурс + метод):

- `POST /createUser` → `POST /users`
- `GET /getUser?id=42` → `GET /users/42`
- `POST /users/42/delete` → `DELETE /users/42`

## Уровни зрелости (Richardson)

Понимать как ориентир, не как догму:

- **0** — один URL, всё через POST (по сути RPC поверх HTTP).
- **1** — появились ресурсы (разные URL).
- **2** — правильно используются методы и коды статуса. **Большинство «REST»
  API живут здесь** — и это нормально.
- **3** — HATEOAS: в ответах ссылки на доступные действия. На практике редко.

## REST vs RPC

- **REST** — вокруг ресурсов и стандартных методов; хорош для CRUD и публичных
  API.
- **RPC** (например gRPC) — вокруг **вызова процедур** (`getUser`,
  `transfer`); удобнее для операций-действий и межсервисного общения.

## Как ответить на интервью

Коротко: REST — архитектурный стиль, не протокол. Работаем с ресурсами по URL,
действия выражаем HTTP-методами, состояние отдаём представлениями (JSON).
Принципы: ресурсо-ориентированные URL (существительные, не глаголы),
единообразный интерфейс из стандартных методов и кодов, stateless-запросы.
Большинство реальных API — это уровень 2 по Ричардсону (ресурсы + правильные
методы/коды), HATEOAS почти не встречается. В отличие от RPC, где мы вызываем
процедуру, REST крутится вокруг сущностей.
