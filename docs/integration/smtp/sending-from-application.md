# Отправка писем из приложения

С точки зрения бэкенда отправка письма — это обращение к SMTP-серверу. В Spring
для этого есть `JavaMailSender`. Но главное на проде — не сам вызов, а
**как встроить отправку надёжно**.

## Базовая отправка в Spring

```java
@Service
class MailService {
    private final JavaMailSender sender;

    void sendWelcome(String to) {
        SimpleMailMessage msg = new SimpleMailMessage();
        msg.setTo(to);
        msg.setSubject("Добро пожаловать");
        msg.setText("Привет!");
        sender.send(msg);
    }
}
```

Настройки SMTP (хост, порт, логин, TLS) — в конфиге приложения. Для HTML-писем
и вложений — `MimeMessage` + `MimeMessageHelper`. Часто тело письма рендерят
шаблонизатором (Thymeleaf).

## Почему не слать письмо синхронно в запросе

Отправка через внешний SMTP — **медленная и ненадёжная** операция (сеть,
таймауты, недоступность). Если слать прямо в HTTP-запросе:

- пользователь ждёт ответа, пока идёт отправка;
- падение SMTP роняет бизнес-операцию (например, регистрацию);
- нет ретраев — письмо просто теряется.

## Надёжный подход

- **Асинхронно / через очередь** — бизнес-операция завершается сразу, а письмо
  уходит в фон (очередь задач, Kafka, `@Async`). Провайдер тупит — не ломает
  основной поток.
- **Ретраи** с backoff на временные ошибки.
- **Идемпотентность** — не слать одно и то же письмо дважды при повторе (ключ/
  отметка «отправлено»).
- **Отдельный сервис/провайдер** рассылок (транзакционная почта) — берёт на
  себя доставляемость, шаблоны, аналитику; приложение только формирует событие
  «отправить письмо».

## Как ответить на интервью

Коротко: в Spring письма шлют через `JavaMailSender` (`MimeMessage` для HTML и
вложений, тело часто рендерят Thymeleaf), настройки SMTP — в конфиге. Но
ключевое — не слать письмо синхронно в HTTP-запросе: внешний SMTP медленный и
ненадёжный, иначе пользователь ждёт, а падение почты роняет бизнес-операцию.
Правильно — отправлять асинхронно или через очередь, с ретраями и
идемпотентностью, а часто вообще делегировать отдельному сервису
транзакционной рассылки, который отвечает за доставляемость.
