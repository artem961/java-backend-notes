# Методы и коды ответа

Метод говорит, **что сделать** с ресурсом; код ответа — **чем закончилось**.
Правильный выбор метода и кода — половина хорошего REST API.

## Методы

| Метод | Смысл | Идемпотентен | Безопасен |
|---|---|---|---|
| `GET` | получить ресурс | да | да (не меняет данные) |
| `POST` | создать / выполнить действие | нет | нет |
| `PUT` | заменить ресурс целиком | да | нет |
| `PATCH` | частично изменить | не обязательно | нет |
| `DELETE` | удалить | да | нет |

- **Безопасный (safe)** — не меняет состояние на сервере (`GET`, `HEAD`).
- **Идемпотентный** — повтор запроса даёт тот же результат, что и один раз.
  `PUT`/`DELETE` идемпотентны (удалить дважды — итог тот же), `POST` — нет
  (два `POST` создадут две записи).

## Классы кодов

- **1xx** — информационные (редко используются напрямую).
- **2xx** — успех: `200 OK`, `201 Created` (создан ресурс), `204 No Content`
  (успех без тела).
- **3xx** — перенаправление: `301` (постоянно), `302`/`307` (временно),
  `304 Not Modified` (кэш актуален).
- **4xx** — ошибка клиента: `400` (плохой запрос), `401` (не
  аутентифицирован), `403` (нет прав), `404` (не найдено), `409` (конфликт),
  `422` (не прошёл валидацию), `429` (слишком много запросов).
- **5xx** — ошибка сервера: `500` (внутренняя), `502`/`503`/`504` (проблемы
  шлюза/недоступность/таймаут).

## Частые ловушки

- **401 vs 403**: 401 — «не знаю, кто ты» (нет/неверные креды), 403 — «знаю, но
  тебе нельзя».
- **200 с ошибкой в теле** — антипаттерн: код должен отражать результат,
  ошибку нельзя прятать под `200 OK`.
- **POST для чтения** — плохо: `GET` кэшируется, безопасен и идемпотентен.

## Как ответить на интервью

Коротко: метод описывает намерение — `GET` читает, `POST` создаёт, `PUT`
заменяет целиком, `PATCH` меняет частично, `DELETE` удаляет. Ключевые
свойства: `GET` безопасен (не меняет данные), а `GET`/`PUT`/`DELETE`
идемпотентны — повтор безопасен, тогда как `POST` нет. Коды: 2xx — успех,
3xx — редирект/кэш, 4xx — вина клиента (401 — не аутентифицирован, 403 — нет
прав, 404 — нет ресурса, 409 — конфликт), 5xx — вина сервера. Главное — код
должен честно отражать исход, а не отдавать 200 с ошибкой в теле.
