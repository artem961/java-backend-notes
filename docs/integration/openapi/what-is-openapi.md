# Что такое OpenAPI

OpenAPI — стандарт **описания REST API** в машиночитаемом виде (обычно
YAML/JSON). Это формальный контракт: какие есть эндпоинты, методы, параметры,
форматы запросов/ответов и коды. **Swagger** — набор инструментов вокруг
OpenAPI (в том числе Swagger UI).

## OpenAPI и Swagger — не путать

- **OpenAPI** — сам стандарт/спецификация (раньше называлась Swagger
  Specification).
- **Swagger** — инструменты: **Swagger UI** (интерактивная документация),
  Swagger Editor, генераторы. Часто «сваггером» по привычке зовут и саму
  документацию.

## Что описывает

- **Пути и методы**: `GET /users/{id}`, `POST /users`.
- **Параметры**: path/query/header, обязательность, типы.
- **Тела запросов и ответов**: схемы данных (JSON Schema).
- **Коды ответов** и их форматы, включая ошибки.
- **Аутентификацию**: схемы безопасности (Bearer, OAuth2).

## Зачем нужен контракт

- **Единый источник правды** о том, как устроено API — для фронта, мобилки,
  внешних интеграторов.
- **Документация** — Swagger UI рендерит спецификацию в интерактивную страницу,
  где можно посмотреть и подёргать эндпоинты.
- **Кодогенерация** — по спецификации генерят клиентов и серверные заготовки на
  разных языках.
- **Тесты и валидация** — можно проверять, что реальные ответы соответствуют
  контракту.

## В Spring

Библиотека **springdoc-openapi** автоматически строит OpenAPI-спецификацию из
контроллеров и DTO и поднимает Swagger UI. Это **code-first** подход (см.
соседнюю тему): спецификация генерится из кода.

## Как ответить на интервью

Коротко: OpenAPI — стандарт машиночитаемого описания REST API: эндпоинты,
методы, параметры, схемы запросов/ответов, коды, аутентификация. Swagger — это
инструменты вокруг него, в первую очередь Swagger UI с интерактивной
документацией. Контракт даёт единый источник правды, живую документацию,
кодогенерацию клиентов и серверных заготовок и возможность валидировать ответы.
В Spring его обычно генерируют из кода через springdoc-openapi (code-first) и
показывают в Swagger UI.
